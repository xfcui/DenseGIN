# Ring Size Feature

## What is encoded

Both atom features and bond features receive one extra index: the **minimum ring
size** of the smallest ring that contains the atom or bond, encoded by
`possible_ring_size_list = [3, 4, 5, 6, 7, 8, 'misc']`.

- **Atom**: `RingInfo.MinAtomRingSize(atom_idx)` — smallest SSSR ring the atom
  belongs to, or `0` if the atom is not in any ring.
- **Bond**: `RingInfo.MinBondRingSize(bond_idx)` — smallest SSSR ring the bond
  belongs to, or `0` if the bond is not in any ring.

Both use `safe_index`, so anything outside the explicit list (i.e. `0` for
acyclic atoms/bonds, and rings of size ≥ 9) falls into the `'misc'` bucket at
the last index.

## Why minimum ring size, not maximum or average

Ring membership is multi-valued — an atom can sit at the junction of a 5-ring
and a 6-ring simultaneously (e.g., indene, azulene).  Among all ring sizes the
**minimum** is most discriminative because:

- It determines whether an atom is in a strained ring (3 or 4), which strongly
  influences bond angles, hybridization, and reactivity.
- At fused junctions, the smaller ring dominates geometry.
- The largest or average ring size of a fused system is already partially
  captured by the atom's degree and hybridization features that are already
  present.

For bonds the same reasoning applies: bridgehead and fused bonds belong to two
rings; the smaller ring is the tighter geometric constraint.

## Why a discrete vocabulary [3–8] with `misc`

- **3–8 covers the overwhelming majority of drug-like molecules.**  
  The distribution plots (`ring_stats_vs_heavy_10k.svg`) show that max ring size
  reaches 14 at the extremes, but the median max ring size over 10,000 PCQM4Mv2
  molecules is 6.  Only a tiny fraction exceed size 8 (macrocycles).
- **Discrete embedding is consistent with all other categorical features**
  in this codebase (atomic number, degree, formal charge, …).  A continuous
  integer would break the integer-index assumption used everywhere.
- **Rings of size 0** (acyclic) and **rings of size ≥ 9** (macrocycles) share
  the `misc` bucket.  They carry very different chemistry, but both are
  infrequent enough that the model can distinguish them using the companion
  `is_in_ring` boolean that is already present in the atom feature vector.

## Interaction with existing features

| Existing feature | What it covers | What ring size adds |
|---|---|---|
| `is_in_ring` (atom) | Binary: ring or not | Granular size of the ring |
| `bond_type` (AROMATIC) | Aromaticity | Small aromatic rings (5 vs 6) |
| `hybridization` (SP2/SP3) | Local geometry | Strained rings (3/4) force SP3 but differ from open-chain SP3 |
| `is_conjugated` (bond) | π system | Ring conjugation vs chain conjugation |

Ring size is **not redundant** with any of these individually, and it is
particularly important for HOMO-LUMO gap prediction because the LUMO energy of
conjugated systems is sensitive to ring strain and ring size (e.g., fulvene vs
benzene vs cyclopentadienyl).

## RDKit API used

```python
ri = mol.GetRingInfo()
ri.MinAtomRingSize(atom_idx)  # -> int, 0 if acyclic
ri.MinBondRingSize(bond_idx)  # -> int, 0 if not in ring
ri.NumRings()                 # used in distribution plots only
ri.AtomRings()                # list of tuples; len(r) gives each ring size
```

All calls are on the post-`AddHs` / post-sanitization molecule, so ring
membership is consistent with aromaticity and valence perception.

|---|---|---|

| `ring_stats_vs_heavy_10k_max_ring_size_vs_heavy.svg` | heavy atom count | max ring size |

Key statistics on 10,000 random PCQM4Mv2 molecules:

| Statistic | Value |
|---|---|
| Acyclic molecules (no rings) | 1,193 / 10,000 (11.9%) |
| Median ring count | 1 |
| Max ring count | 7 |
| Median max ring size | 6 |
| Max max ring size | 14 |
| Median heavy atom count | 15 |
