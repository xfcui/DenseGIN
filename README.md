# PCQM4Mv2 HOMO-LUMO Gap Prediction

JAX/Equinox/Optax pipeline for predicting HOMO-LUMO gaps on the
[PCQM4Mv2](https://ogb.stanford.edu/docs/lsc/pcqm4mv2/) molecular graph
benchmark. The target is measured in eV and evaluated with MAE.

The repository contains:

- `src.dataset.dataprocess`: raw SMILES plus optional SDF coordinates -> HDF5 graph tensors.
- `src.dataset`: cached HDF5 dataset loading and padded collapsed graph batching.
- `src.model`: `DuAxMPNN`, an Equinox message-passing model with multi-hop edges.
- `src.train`: training loop, validation loop, Adan optimizer setup, and LR scheduling.

## Quick Start

```bash
# Install runtime dependencies.
python -m pip install -r requirements.txt

# dataprocess.py imports pandas; install it if your environment does not already provide it.
python -m pip install pandas

# Build processed graph tensors.
python -m src.dataset.dataprocess --h-mode active

# Train and save the best validation checkpoint.
python -m src.train --model_save_path results/best_model.eqx

# Run smoke/regression tests.
python -m pytest tst/ -q
```

## Data Layout

By default, commands resolve paths from the current working directory and expect this layout:

```text
dataset/
  pcqm4m-v2/
    raw/
      data.csv.gz              # columns: smiles, homolumogap
      pcqm4m-v2-train.sdf      # optional 3D coordinates
    split_dict.h5              # train/valid/test split indices
    processed/
      data_processed.h5        # generated by dataprocess.py
```

Preprocessing writes one processed HDF5 file. The dataset loader expects
`split_dict.h5` to already exist under `dataset/pcqm4m-v2/`.

Hydrogen handling is controlled by `--h-mode`:

- `active` -> `processed/data_processed.h5` (default)
- `all` -> `processed/data_processed_all.h5`
- `heavy` -> `processed/data_processed_heavy.h5`

Use `--overwrite` when regenerating an existing processed file.

```bash
python -m src.dataset.dataprocess \
  --dataset-root dataset \
  --dataset-name pcqm4m-v2 \
  --h-mode active \
  --overwrite
```

## Training

```bash
python -m src.train \
  --batch_size 512 \
  --learning_rate 3e-3 \
  --weight_decay 2e-2 \
  --scheduler_period 8 \
  --model_save_path results/best_model.eqx \
  --seed 0
```

The CLI trains for `scheduler_period * 8` epochs. With the default
`--scheduler_period 8`, that is 64 epochs. The scheduler warms up for one
period, holds the peak learning rate for one period, then follows geometric
cosine annealing. The best checkpoint by validation MAE is serialized with
`eqx.tree_serialise_leaves`.

Common training arguments:

| Argument | Default | Description |
|---|---:|---|
| `--batch_size` | `512` | Real graphs per batch. A null graph is prepended internally for padding. |
| `--learning_rate` | `3e-3` | Peak learning rate. |
| `--weight_decay` | `2e-2` | Base weight decay before per-parameter multipliers. |
| `--scheduler_period` | `8` | Period length `k`; total epochs are `k * 8`. |
| `--model_save_path` | `results/best_model.eqx` | Best-checkpoint output path. |
| `--seed` | `0` | RNG seed for model init, training shuffle, and dropout. |
| `--h-mode` | `active` | Chooses the processed HDF5 variant unless `--processed-h5` is set. |
| `--processed-h5` | `None` | Explicit processed HDF5 basename under `processed/`. |

Ablation flags are exposed directly from `AblationConfig`:

```bash
python -m src.train \
  --max_hops 4 \
  --depth_mode dense \
  --cont_embed moact \
  --moact_bases 8 \
  --elec_mode absolute
```

Use `--no_neighbor_rank` to replace the direct-edge neighbor-rank feature with
a neutral token.

## Model

`get_model()` creates `DuAxMPNN(depth=5, width=256, num_head=16, dim_head=16)`.
The forward pass consumes batches produced by `PCQMDataloader`.

Key components:

- `EmbedLayer` embeds 10 discrete atom features with vocabulary offsets applied by the dataset.
- `GatedLinearBlock` projects RWPE positional features and performs gated MLP-style mixing.
- `LayerMixerKernel` combines 1-hop through `max_hops` bond-aware convolutions with virtual-node context.
- `DepthMixerKernel` optionally performs dense cross-layer aggregation.
- `HeadKernel` sum-pools graph nodes, drops the null graph, and predicts one scalar gap per molecule.

The training loss is MAE between predicted and target HOMO-LUMO gaps.

## Dataset API

```python
from src.dataset import PCQMDataset, PCQMDataloader

dataset = PCQMDataset(
    dataset_root="dataset/pcqm4m-v2",
    split="train",
    load_in_memory=True,
)

loader = PCQMDataloader(
    dataset,
    batch_size=512,
    shuffle=True,
    drop_last=True,
    seed=0,
)

for batch in loader:
    node_feat = batch["node_feat"]      # int32, shape (N, 10)
    node_embd = batch["node_embd"]      # float32, shape (N, 17)
    edge_idx = batch["edge_index"]      # int32, shape (2, E)
    labels = batch["labels"]            # float32, shape (B,)
    break
```

`PCQMDataset` instances that point to the same processed file share an
in-process cache. Split indices are loaded from `split_dict.h5`, and molecules
with `label < 0` are filtered out during dataset construction.

The functional `batch_collapse(dataset, graph_ids, *, pad_to_multiple=1024)` is
also exported for direct collation.

`PCQMDataloader(..., pad_to_multiple=None)` pads to `batch_size * 4`; pass an
explicit value to pin node and edge row counts to a fixed multiple such as 1024.

## Batch Format

Collapsed batches concatenate multiple molecules into one padded graph block.
Graph index `0` is a zero-filled null graph that absorbs padding; real graph
indices start at `1`. Node and edge rows are padded independently to the active
`pad_to_multiple`.

Core keys:

| Key | dtype | Shape | Notes |
|---|---|---|---|
| `node_feat` | `int32` | `(N, 10)` | Discrete atom features with offsets applied. |
| `node_embd` | `float32` | `(N, 17)` | Continuous features: RWPE, geometry, electronegativity, charges. |
| `node_ptr` | `int32` | `(B + 2,)` | Node boundaries for null plus real graphs. |
| `node_batch` | `int32` | `(N,)` | Graph index per node. |
| `edge_index` | `int32` | `(2, E)` | Direct-bond edge indices with graph offsets applied. |
| `edge_feat` | `int32` | `(E, 6)` | Direct-bond features with offsets applied. |
| `edge_2hop_*` | mixed | varies | 2-hop edge index, feature, ptr, and batch arrays. |
| `edge_3hop_*` | mixed | varies | 3-hop edge index, feature, ptr, and batch arrays. |
| `edge_4hop_*` | mixed | varies | 4-hop edge index, feature, ptr, and batch arrays. |
| `labels` | `float32` | `(B,)` | HOMO-LUMO gap in eV. |
| `molecule_ids` | `int64` | `(B,)` | Original dataset row indices. |
| `batch_n_graphs` | `int32` | scalar | Number of real graphs, excluding the null graph. |

On disk, HDF5 stores compact arrays (`uint8` features/indices, `float16`
continuous features, `int32` pointers). The loader returns standard training
dtypes (`int32` and `float32`).
